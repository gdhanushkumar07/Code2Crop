# 🌾 Code2Crop

### AI Intelligence for Every Farmer

Code2Crop is an AI-powered agricultural advisory platform that helps smallholder farmers make data-driven crop decisions using weather forecasting, climate history, and generative AI — delivered through a farmer-first web app **and** WhatsApp, with a human expert safety net for every uncertain AI call.

Built for **Google Build with AI — Code for Communities**.

---

## 📌 The Problem

Farmers in drought-affected regions like Karimnagar, Peddapalli, and Siddipet (Telangana) traditionally choose what to plant based on ancestral habit or local hearsay rather than current soil, water, and weather conditions. This leads to:

- Heavy **groundwater depletion** from water-intensive crops planted in the wrong season
- **Failed yields** and high electricity costs from unnecessary pump irrigation
- Generic, plain-text government SMS advisories that farmers routinely ignore
- Agricultural chatbots that confidently state a diagnosis even when they shouldn't — a dangerous failure mode when the stakes are a farmer's entire harvest

## 💡 The Solution

Code2Crop unifies weather data, climate history, AI reasoning, multilingual voice interaction, and real human experts into one platform, accessible however a farmer can reach it — smartphone app or WhatsApp.

### Core Capabilities

| Capability | What it does |
|---|---|
| 🧭 **Smart Crop Recommendation** | Ranks crops using weather data, computed NDVI, soil moisture estimates, and rainfall history, weighing each crop's water need against the local groundwater table |
| ☀️ **Predictive Weather Intelligence** | Flags dry spells before they hit and pushes localized-language irrigation advisories |
| 🛰️ **Weather & Soil Intelligence** | Computes NDVI, soil moisture, groundwater deficit, and AQI from Open-Meteo weather models — displayed as live telemetry stats in the farmer's Home Hub |
| 🍃 **AI Disease Diagnosis** | Gemini 1.5 Flash Vision analyzes leaf/stem photos, cross-referenced with voice-described symptoms |
| 🎙️ **Voice-First Experience** | Speech-to-text (Groq Whisper) and Edge TTS so non-literate farmers can simply speak and listen in their native dialect |
| 👨‍🌾 **Human Expert Escalation** | Any AI diagnosis under a 65% confidence threshold is auto-routed — with photo and GPS — to the nearest Rythu Seva Kendra (RSK) officer |

## 🖥️ Two Portals, One Platform

### 🌱 Farmer Portal (`/farmer`)

A warm, simple interface built for everyday use:

- **Home Hub** — live weather telemetry (Open-Meteo), soil moisture index, groundwater deficit, NDVI, dry-spell alert banner, interactive Leaflet map showing the farmer's plot
- **Recommendations** — side-by-side crop comparison with suitability %, water savings, projected yield, estimated profit, and soil health — generated by Gemini 3.1 Flash Lite with real climate context
- **AI Chat** — type, speak (voice recorder with animated waveform), or upload a leaf photo in one thread, with the AI's 6-step reasoning pipeline visualized live (Multilingual Voice → STT → Translation → Gemini Orchestrator → Grounding Layers → Advisory Synthesis)
- **Language Selector** — 6 Indian languages (Telugu, Hindi, Kannada, Tamil, Marathi, English)
- **WhatsApp Linking** — connect your Google account to your WhatsApp phone number for a seamless cross-platform experience

### 🛰️ RSK Officer Portal (`/rsk`) — Operations Center

A dark, dashboard-style command center for field officers:

- **Admin Authentication** — Google Sign-In with Firestore role check (`anachipravalgupta@gmail.com` as admin). Non-admins see a restricted view without case details, chat logs, or action buttons.
- **Case Queue** — live-updating list of escalated cases with severity filtering and search, polling every 8 seconds
- **Interactive Map** — Leaflet map with 3 base layers (hotspot/dark, satellite via Esri, topographical via OpenTopoMap), severity-coded markers (red/amber/green) with pulsing animations for high-severity cases, click-to-select
- **Case Detail Drawer** — farmer info, GPS coordinates, crop, AI diagnosis with confidence score, uploaded image, and full AI chat transcript
- **Decision & Response Broadcast** — type an advisory reply sent directly via WhatsApp Twilio. First reply auto-sets the case to `in_progress`, which disables further AI auto-responses for that farmer.
- **Approve & Resolve** — sends a localized closure message (auto-detected Telugu/Hindi/Kannada/Tamil/Marathi/English) and marks the case resolved
- **Notifications Feed** — real-time alerts for new escalations
- **Telemetry** — queue size, pending cases count

### 💬 WhatsApp Channel

A Twilio webhook (`POST /api/whatsapp/webhook`) lets farmers without the app text, send voice notes, or share photos directly on WhatsApp:

- **Onboarding Flow** — 4-step conversation: NAME → LANGUAGE → LOCATION → CROP, saved to Firestore profile
- **Audio Transcription** — voice notes downloaded from Twilio MediaUrl, transcribed via Groq Whisper-large-v3
- **Image Persistence** — WhatsApp images downloaded and saved to `public/uploads/` for permanent access (vs. Twilio's short-lived MediaUrl)
- **AI Response** — Gemini generates an agronomist-grade reply in the farmer's chosen language, grounded on weather, climate history, and chat context
- **TTS Voice Reply** — if the farmer sent audio, the AI reply is also synthesized via Edge TTS and sent as a voice message
- **Escalation Auto-Route** — disease keywords in the AI response trigger automatic case creation in the RSK queue
- **Admin Chat Mode** — when a case has active RSK replies (`rskReplyCount > 0`), the AI auto-response is disabled; only the RSK officer can reply until the case is resolved

## 🎙️ Audio Pipeline

### Speech-to-Text (Farmer → AI)

```
Voice Recording (Web) / Twilio Audio (WhatsApp)
    → POST /api/chat/transcribe → Groq Whisper-large-v3
    → Auto language detection (te, hi, kn, ta, mr, en)
    → Google Translate (non-English → English)
    → Sent to Gemini with original text preserved
```

### Text-to-Speech (AI → Farmer)

```
Gemini response (English or native language)
    → GET /api/chat/speech?text=...&lang=...
    → Edge TTS WebSocket (speech.platform.bing.com)
    → 6 Indian voices via VOICE_MAP
    → MP3 audio served with Cache-Control headers
    → WhatsApp: saved to public/tts/, sent as Twilio media message
    → Web: played via VoiceMessagePlayer component
```

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- A Google Gemini API key
- A Groq API key (for Whisper speech-to-text)
- A Firebase project with a service account (Firestore + Auth)
- A Twilio account with WhatsApp Sandbox/Business API access

### Installation

```bash
git clone https://github.com/gdhanushkumar07/Code2Crop.git
cd Code2Crop
npm install
```

### Environment Variables

Create a `.env.local` file in the project root:

```env
# Google Gemini
GOOGLE_GEMINI_API_KEY=your_gemini_api_key

# Groq (Whisper STT)
GROQ_API_KEY=your_groq_api_key

# Firebase Admin
FIREBASE_PROJECT_ID=your_firebase_project_id
FIREBASE_CLIENT_EMAIL=your_firebase_client_email
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"

# Firebase Client (Next.js public vars for frontend auth)
NEXT_PUBLIC_FIREBASE_API_KEY=your_firebase_api_key
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_firebase_project_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_firebase_app_id

# Twilio (WhatsApp)
TWILIO_ACCOUNT_SID=your_twilio_account_sid
TWILIO_AUTH_TOKEN=your_twilio_auth_token
TWILIO_WHATSAPP_NUMBER=whatsapp:+14155238886

# Public base URL (for TTS audio on WhatsApp — must not be localhost)
NEXT_PUBLIC_BASE_URL=https://your-deployed-url.vercel.app
```

### Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the landing page. Visit `/farmer` for the Farmer Portal and `/rsk` for the RSK Operations Center.

> **Note**: Twilio WhatsApp audio replies (TTS) require a publicly accessible URL. Local development with `localhost` will skip audio and send text-only replies.

### WhatsApp Webhook Setup

Point your Twilio WhatsApp sandbox/number's webhook to:

```
POST https://<your-deployed-url>/api/whatsapp/webhook
```

Configure your Twilio number's messaging webhook to use HTTP POST for incoming messages.

| Role | Can See |

## 🔁 Case Lifecycle

```
                ┌─────────────┐
                │   PENDING   │ ← Auto-created by AI on low-confidence / disease detection
                └──────┬──────┘
                       │
              ┌────────▼────────┐
              │ Admin sends     │── First reply → AI auto-response disabled
              │ first reply     │── Status → in_progress
              └────────┬────────┘
                       │
              ┌────────▼────────┐
              │ Admin clicks    │── Localized closure message sent via WhatsApp
              │ Resolve         │── Status → resolved
              └─────────────────┘
```
